# Abodom
 
Sitio web corporativo desarrollado como proyecto freelance y desplegado en producción. Fue uno de mis primeros proyectos en React, donde opté por configurar el entorno manualmente con Webpack y Babel en lugar de usar un scaffolding como Create React App.
 
---
 
## Sobre el proyecto
 
El cliente necesitaba un sitio web corporativo. El proyecto se entregó y desplegó exitosamente. Posteriormente el cliente migró a WordPress para poder gestionar el contenido de forma autónoma sin depender de un desarrollador — una decisión que me enseñó a valorar la autonomía del cliente como criterio de arquitectura desde el inicio del proyecto.
 
## Stack tecnológico
 
- **React** — librería principal de UI
- **Webpack** — bundler configurado manualmente
- **Babel** — transpilación configurada manualmente
- **SCSS** — preprocesador CSS
## Lo que diferencia este proyecto
 
A diferencia de proyectos bootstrapped con CRA o Vite, aquí configuré el pipeline de build desde cero: entrada/salida de Webpack, loaders para JSX y SCSS, y configuración de Babel. Fue un ejercicio que me dio una comprensión más profunda de cómo funcionan las herramientas bajo el capó antes de abstraerlas.
 
## Contexto
 
Este es uno de mis primeros proyectos en React. El código refleja ese momento de aprendizaje, pero lo incluyo como evidencia de progresión técnica — la diferencia con proyectos posteriores como [Mundo Eléctrico](https://github.com/JulioMartinezTech/mundoelectrico-nuxt) habla por sí sola.
